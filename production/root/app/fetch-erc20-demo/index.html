protocol lifecycle

How the simulated thesis vault works

Execution path shown by the auto-running demo. This is the intended v0 SPOT primitive flow, represented in browser-local state.

1. Factory validates

Quote asset, spot assets, adapters, oracle freshness, weights, caps, and uniqueness are checked before vault creation.

2. Vault launches

Hyper Privacy vault and share token become the selected thesis object for the local protocol surface.

3. Buy deposits

Mock mUSDC enters the vault, fees are split, shares mint only against added NAV, and spot composition is allocated.

4. Rebalance

Creator-gated rebalance sells to quote and redeploys into approved SPOT adapters with fresh oracle pricing.

5. Sell redeems

Shares burn, spot exposure exits to quote, user receives mock mUSDC, and holder fee remains in NAV.
